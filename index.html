To create and deploy your AI-generated website, we need access to your GitHub account. Below you'll find detailed explanations of each permission we're requesting.

Required Permissions

Repository Access repo

Create and manage your website repositories

Why we need this:

  • Create a new repository for your generated website
  • Upload your website files and assets
  • Manage repository settings for GitHub Pages deployment
  • Update your website content when you make changes

What we can do: Read and write access to your repositories

What we cannot do: Access private repositories unless explicitly granted

Learn more about repo scope

GitHub Actions workflow

Enable GitHub Actions for automatic deployments

Why we need this:

  • Set up automated deployment workflows
  • Enable continuous integration for your website
  • Automatically build and deploy changes
  • Monitor deployment status and logs

What we can do: Create, update, and manage GitHub Actions workflows

What we cannot do: Access workflow secrets or sensitive data

Learn more about workflow scope

Email Access user:email

Set up GitHub Pages with your email

Why we need this:

  • Configure GitHub Pages with proper attribution
  • Set up commit author information
  • Enable notifications for deployment status
  • Comply with GitHub's terms of service

What we can do: Read your primary and verified email addresses

What we cannot do: Send emails on your behalf or access private emails

Learn more about user:email scope

Your Security & Privacy

  • Secure OAuth: We use GitHub's official OAuth flow for authentication
  • No Password Storage: We never see or store your GitHub password
  • Revocable Access: You can revoke these permissions at any time in your GitHub settings
  • Limited Scope: We only request the minimum permissions needed for functionality
