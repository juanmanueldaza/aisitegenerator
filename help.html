What are GitHub Permissions?

GitHub permissions (also called "scopes") define what actions an application can perform on your behalf. When you authorize AI Site Generator to access your GitHub account, you're granting specific, limited permissions that allow us to create and manage your website repositories.

Why We Need These Permissions

AI Site Generator needs to interact with your GitHub account to:

  • Create a new repository for your generated website
  • Upload your website files and assets
  • Configure GitHub Pages for automatic hosting
  • Set up automated deployment workflows
  • Manage repository settings and configurations
Good to know: We only request the minimum permissions needed for our service to function. You can revoke these permissions at any time through your GitHub settings.

Detailed Scope Explanations

Repository Access (repo)

Official GitHub Description: Grants full access to repositories, including private repositories.

What AI Site Generator Can Do:

  • Create new public repositories for your websites
  • Read and write files in repositories we create
  • Manage repository settings (like enabling GitHub Pages)
  • Create and manage branches
  • Handle commits and version control

What AI Site Generator Cannot Do:

  • Access your existing private repositories (unless you explicitly choose to use them)
  • Delete repositories without your explicit consent
  • Modify repositories not created by our application
  • Access sensitive data in other repositories
Example GitHub API calls we might make: POST /user/repos (create a new repository) PUT /repos/{owner}/{repo}/contents/{path} (upload website files) PATCH /repos/{owner}/{repo} (enable GitHub Pages)

Workflow Access (workflow)

Official GitHub Description: Grants the ability to add and update GitHub Actions workflow files.

What AI Site Generator Can Do:

  • Create GitHub Actions workflows for automatic deployment
  • Update deployment configurations
  • Monitor workflow execution status
  • Configure continuous integration for your website

What AI Site Generator Cannot Do:

  • Access workflow secrets or environment variables
  • Modify workflows in other repositories
  • Access sensitive build logs or artifacts
  • Execute arbitrary code outside of our defined workflows
Example workflow we might create: name: Deploy Website on: push: branches: [ main ] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Deploy to GitHub Pages uses: peaceiris/actions-gh-pages@v3

Email Access (user:email)

Official GitHub Description: Grants read access to a user's email addresses.

What AI Site Generator Can Do:

  • Read your primary email address
  • Access verified email addresses
  • Use your email for proper git commit attribution
  • Configure GitHub Pages with correct contact information

What AI Site Generator Cannot Do:

  • Send emails on your behalf
  • Access private or unverified email addresses
  • Share your email with third parties
  • Subscribe you to mailing lists
Privacy Note: We use your email address only for technical purposes like git commits. We never share your email address with third parties or use it for marketing purposes.

Security & Privacy

OAuth 2.0 Security

AI Site Generator uses GitHub's official OAuth 2.0 implementation, which is the industry standard for secure authorization. This means:

  • No Password Access: We never see or store your GitHub password
  • Secure Token Exchange: All communication is encrypted and uses secure tokens
  • Limited Lifetime: Access tokens can expire and be revoked
  • Scope Limitation: We can only perform actions within the granted scopes

Data Protection

  • We only access data necessary for our service functionality
  • No sensitive data is stored unnecessarily
  • All data transmission uses HTTPS encryption
  • Access tokens are stored securely and never exposed

Compliance

Our OAuth implementation follows GitHub's security guidelines and best practices:

  • CSRF protection with state parameters
  • Secure redirect URI validation
  • Proper token storage and handling
  • Regular security audits and updates

Managing Your Permissions

Revoking Access

You can revoke AI Site Generator's access to your GitHub account at any time:

  1. Go to your GitHub Settings → Applications
  2. Find "AI Site Generator" in the "Authorized OAuth Apps" section
  3. Click "Revoke" to remove all permissions
Note: Revoking access will not delete repositories or websites that have already been created. It only prevents future access to your GitHub account.

Monitoring Activity

You can monitor what applications are doing with your account:

  • Check the Security Log in your GitHub settings
  • Review the audit log for OAuth application activities
  • Monitor repository creation and modification events

Re-authorizing

If you revoke access and later want to use AI Site Generator again, you'll need to go through the authorization process again. This ensures you maintain full control over your permissions.

Troubleshooting

Authorization Failed

Problem: The authorization process fails or returns an error.

Solutions:

  • Make sure you're logged into GitHub
  • Check that you haven't already denied access recently
  • Try clearing your browser cache and cookies
  • Ensure JavaScript is enabled in your browser

Missing Permissions

Problem: AI Site Generator can't perform certain actions.

Solutions:

  • Check that you granted all required scopes during authorization
  • Verify that your GitHub account has the necessary permissions
  • Try re-authorizing with all required permissions

Repository Creation Issues

Problem: New repositories aren't being created.

Solutions:

  • Check if you've reached GitHub's repository limit
  • Verify that the repository name isn't already taken
  • Ensure you have the "repo" scope authorized

GitHub Pages Not Working

Problem: Your website isn't deploying to GitHub Pages.

Solutions:

  • Check that GitHub Pages is enabled for your repository
  • Verify that the workflow scope is authorized
  • Check the Actions tab in your repository for deployment status
  • Ensure your repository has a valid index.html file
Still having issues? If these solutions don't help, please check our GitHub Issues page or create a new issue with details about your problem.
